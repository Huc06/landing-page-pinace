## What an agent will do on Cetus

- **Spot swaps** through Cetus's CLMM pools (`router::swap_in`).
- **LP position management** — open / close / rebalance positions
  bounded by spending limit + slippage policy.

## Policies that apply

- `spending_limit_policy` — required.
- `slippage_guard_policy` — strongly recommended; CLMM prices move
  fast inside narrow ranges.
- A future `lp_band_policy` — bounds how far the agent can move a
  position from its current range. Spec in progress.
