agent shipped (Fenik) trades on **DeepBook v3**. The next wave of

## Cross-cutting infra

These show up under every integration:

- **zkLogin** (planned) — onboard owners with Google / Apple, no
  seed phrase.
- **Enoki** (planned) — sponsor agent gas so users don't top up the
  agent address themselves.
- **Pyth / Switchboard** — oracle reads for limit-order conditions
  in custom policies.

## Adding a venue

A new venue ships as **three pieces**:

1. **SDK builder** in `@pinace/core` (or a side package): one
   function that takes a `Transaction` + venue args, returns the
   `Coin<U>` your `return_coin` step expects.
2. **Policy mapping**: declare which existing policies gate this
   venue (`spending_limit` is always required; `slippage_guard` and
   `token_whitelist` are common; some venues need a venue-specific
   one).
3. **Wallet adapter** (optional but recommended): a small UI patch
   that lets the wallet's Quote / Plan cards pre-flight against the
   venue's quote API before the user signs.

PRs welcome. Each venue page below documents the action shape and
policy combination Pinace recommends.
